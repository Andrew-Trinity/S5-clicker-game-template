# Marking Guide — Clicker Game

## Outcomes
- **CT5-THI-01**: computational, design, and systems thinking.
- **CT5-DAT-02**: data acquisition, representation, analysis, visualisation.

## Autograder (35%)
- Structure checks (files/elements): **10%**
- Lint quality (JS + HTML): **15%**
- Functionality (increment + reset): **10%**

## Teacher-assessed (65%)
- **Systems thinking (CT5-THI-01)** — *20%*
  - High: Clear inputs, processes, outputs, and feedback; sensible boundaries; accessible design choices justified.
  - Sound: Main components identified; some justification.
  - Developing: Lists parts; minimal rationale.
- **Computational/design thinking (CT5-THI-01)** — *15%*
  - High: Clean state model, event handlers, modular code; commits tell a design story.
  - Sound: Working structure with some refactoring.
  - Developing: Minimal structure; “just works”.
- **Data (CT5-DAT-02)** — *20%*
  - High: Timestamps captured; CPS computed; simple chart; interprets findings in README/notes.
  - Sound: Totals + CPS computed; partial visualisation.
  - Developing: Totals only; no visualisation.
- **Process evidence (Git)** — *10%*
  - High: Frequent, descriptive commits; branches for features; reflection in messages/README.
  - Sound: Regular commits; some descriptions.
  - Developing: Few commits; vague messages.

### Suggested Feedback Prompts
- What did your data tell you about pacing/strategy?
- Which design change improved user performance most? Why?
- How does your system’s feedback loop influence player behaviour?
